# Mock режим

## Описание

Mock режим позволяет включить/выключить доступ к панели тестовых данных на странице пользователя.

## Настройка

### Включение mock режима

1. Создайте файл `.env.local` в корне папки `client/` (если его нет)
2. Добавьте переменную:
   ```
   VITE_MOCK_MODE=true
   ```
3. Перезапустите dev-сервер (`npm run dev`)

### Отключение mock режима

1. Откройте файл `.env.local`
2. Измените значение на:
   ```
   VITE_MOCK_MODE=false
   ```
   или удалите эту строку
3. Перезапустите dev-сервер

### Production

Для production сборки создайте файл `.env.production` с нужным значением:

```
# Отключить тестовые данные в production
VITE_MOCK_MODE=false
```

## Как это работает

- Когда `VITE_MOCK_MODE=true` - на странице "Пользователь" отображается кнопка "▶ Показать тестовые данные"
- Когда `VITE_MOCK_MODE=false` или переменная не установлена - панель тестовых данных скрыта

## Файлы

- `.env.local` - локальная конфигурация (не коммитится в git)
- `.env.example` - пример конфигурации для команды
- `.env.production` - конфигурация для production сборки

## Примечания

- Файл `.env.local` автоматически игнорируется git (см. `.gitignore`)
- После изменения переменных окружения необходимо перезапустить dev-сервер
- Используйте `.env.example` как шаблон для настройки
